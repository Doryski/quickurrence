# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Quickurrence, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please open a private security advisory through GitHub's Security tab.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response timeline

- Acknowledgment within 48 hours
- Status update within 7 days
- Fix release as soon as a patch is validated

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |
